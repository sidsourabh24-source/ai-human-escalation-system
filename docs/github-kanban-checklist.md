# GitHub Kanban Board Checklist

Use this checklist while creating your GitHub Project board.

## Board Setup
- [ ] Create a GitHub Project named: AI-Human-Escalation-MVP
- [ ] Add project description with project objective and 5-week delivery timeline
- [ ] Set visibility (private/public) based on team requirement

## Columns (Status)
- [ ] Backlog
- [ ] In Progress
- [ ] Review
- [ ] Done

## Custom Fields
- [ ] Module (Single select: Module 1, Module 2, Module 3, Module 4, Module 5)
- [ ] Week (Single select: Week 1, Week 2, Week 3, Week 4, Week 5)
- [ ] Priority (Single select: High, Medium, Low)
- [ ] Owner (Text or assignee mapping)

## Labels
- [ ] module-1
- [ ] module-2
- [ ] module-3
- [ ] module-4
- [ ] module-5
- [ ] backend
- [ ] frontend
- [ ] integration
- [ ] documentation
- [ ] bug
- [ ] enhancement

## Milestones
- [ ] Week 1 - Foundation
- [ ] Week 2 - Backend Core
- [ ] Week 3 - Handoff Logic
- [ ] Week 4 - Agent Dashboard
- [ ] Week 5 - Lead + Polish

## Workflow Rules
- [ ] Every issue must have Module + Week + Label
- [ ] Move issue to In Progress only after assignment
- [ ] Move issue to Review only with PR link
- [ ] Move issue to Done only after merge and verification

## Team Rules
- [ ] One feature branch per issue
- [ ] One pull request per issue where possible
- [ ] Mandatory PR template usage
- [ ] Squash merge to keep history clean
